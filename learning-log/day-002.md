# Day 2: Understanding How Web Applications Work

Today I learned about the core components of a web application and how they work together.

## The Basic Flow

User → Frontend → API → Database

### User

The person interacting with the application.

### Frontend

The interface users see and interact with, such as forms, buttons, and dashboards.

### API

Acts as a messenger between the frontend and backend systems. It receives requests and returns responses.

### Database

Stores application data such as user accounts, orders, products, and transactions.

## Why This Matters for Application Support

Understanding how these components interact is important because issues can occur at any layer.

For example, if a user cannot log in:

* The frontend might not be sending the request correctly.
* The API might be unavailable.
* The database might contain incorrect user information.

An Application Support Specialist investigates these possibilities to identify the root cause of the problem.

## Key Takeaways

* Web applications consist of multiple connected components.
* APIs play a central role in communication.
* Databases store the information applications need.
* Troubleshooting requires understanding the entire application flow.
