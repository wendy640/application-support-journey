# Day 3: Understanding Frontend, Backend, and Database

Today, I continued my Application Support learning journey by exploring the roles of the Frontend, Backend, and Database in a web application.

## Frontend

The frontend is the part of an application that users interact with directly.

Examples include:
- Buttons
- Forms
- Navigation menus
- Dashboards

Technologies commonly used:
- HTML
- CSS
- JavaScript
- React

As someone who has worked with HTML, CSS, JavaScript, and React, I already have some familiarity with the frontend layer.

## Backend

The backend contains the business logic of an application.

Responsibilities include:
- Processing requests
- Validating data
- Managing authentication
- Communicating with databases

Common backend technologies:
- Node.js
- Python
- Java
- PHP
- .NET

## Database

The database stores application data.

Examples:
- User accounts
- Products
- Orders
- Transactions

Common databases:
- MySQL
- PostgreSQL
- SQL Server

## How They Work Together

User → Frontend → Backend/API → Database

The frontend sends requests to the backend. The backend processes those requests and communicates with the database. The database returns the requested information, which is then displayed to the user through the frontend.

## Why This Matters for Application Support

Understanding these components helps identify where issues may occur when users report problems.

For example:
- Frontend issue: A button does not work.
- Backend issue: The server fails to process a request.
- Database issue: The required data cannot be found.

## Key Takeaway

Application Support is not only about fixing problems. It requires understanding how different parts of an application interact so that issues can be investigated effectively.
